Authentication

This is a work in progress. For now, client-server sessions are cookie based. When a client makes a first request to the REST API, an anonymous user is created and all further requests will be made on behalf of this user as long as the cookie (named access_token) containing a session id is exchanged on each request.

Also, if the first page you access without a cookie is /v1, the cookie that is set will belong to a pre-filled test user, instead of a newly created anonymous user. Using this cookie in further calls will allow you to use the CURL examples on this test data.
